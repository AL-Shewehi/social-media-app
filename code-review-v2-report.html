تفاصيل القضايا والإصلاحات

High#1 — أنواع غير مُصدَّرة
src/types/database.types.ts
أنواع Profile، Post، Comment، Like كانت مكتوبة بشكل مبعثر في ملفات متعددة بدلاً من أن تكون في ملف مشترك واحد.
✔️ الإصلاح: إنشاء src/types/database.types.ts مع تصدير صريح: Profile، Post، Comment، Like، PostCardProps. تم إزالة any[] بالكامل من القاعدة البرمجية وإضافة currentUserProfile.
High#2 — FeedList: as any للـ profiles
src/features/feed/components/FeedList.tsx
إرسال post.profiles as any إلى PostCard يلغي فحص TypeScript بالكامل ويخفي أخطاء شكل البيانات الفعلية.
✔️ الإصلاح: إنشاء دالة normalizeProfile(profiles: unknown): Profile | null تتعامل مع كلٍّ من إرجاع Supabase كمصفوفة أو كائن، وتم استقبال currentUserProfile كبروب جديد.
High#3 — CommentsDialog: any[] + inline types
src/features/feed/components/CommentsDialog.tsx
كان الملف يعرّف الـ Comment محلياً ويستخدم any[] للـ replies. نفس المشكل في CommentItem.
✔️ الإصلاح: استيراد Comment من الأنواع المشتركة. إزالة جميع الاستخدامات لـ any[]. إضافة user: Profile | null إلى Props المكون لاستخدامه في الـ CommentForm.
Medium#4 — Navbar/Sidebar/CreatePost: types مكررة
src/components/shared/Navbar.tsx + Sidebar.tsx + src/features/feed/components/CreatePost.tsx
ثلاث ملفات تعرّف نفس نموذج الـ user inline: { full_name: string | null; avatar_url: string | null; }. أي تغيير في الجدول يتطلب تعديل 3 أماكن.
✔️ الإصلاح: استيراد Profile | null من @/types/database.types في الملفات الثلاثة. تم إزالة تعريفات الأنواع المكررة بالكامل.
Medium#5 — useAuth: تكرار try/catch
src/features/auth/hooks/useAuth.tsx
الدالتان login و register تحتويان على نفس هيكل try/catch مع إدارة loading و error — انتهك مبدأ DRY.
✔️ الإصلاح: استخراج دالة execute<T>(action: () => Promise<T | null>): Promise<T | null> تقوم بإدارة التحميل والخطأ، ثم login و register تستدعيانها بتمرير دالة الـ action فقط.
Medium#6 — Profile query مكرر
src/app/(main)/layout.tsx + src/app/(main)/page.tsx
كلا الملفين يجلبان profiles لنفس المستخدم بشكل مستقل داخل نفس الطلب، مما يسبب استعلام SQL إضافي غير ضروري.
✔️ الإصلاح: إنشاء getProfile(userId: string) في lib/supabase/server.ts باستخدام React.cache(). الآن layout.tsx و page.tsx يمكنهما استدعاؤها دون تكرار الاستعلام الفعلي.
Medium#7 — react-query: إعدادات مفقودة
src/lib/react-query.tsx
الـ QueryClient لم يحدد refetchOnWindowFocus (افتراضياً false) وكان staleTime 60 ثانية طويلاً جداً للبيانات الاجتماعية.
✔️ الإصلاح: تعيين refetchOnWindowFocus: true و staleTime: 30_000 (30 ثانية) لتحديث التغذية عند عودة المستخدم.
Low#8 — PostCard: جمع "تعليق/تعليقات" + User Profile
src/features/feed/components/PostCard.tsx
النص "تعليق" يظهر دائماً بصيغة المفرد حتى لو كان العدد 10. كما أن CommentsDialog لم يستقبل معلومات المستخدم الحالي لعرض الصورة الرمزية في CommentForm.
✔️ الإصلاح: استخدام {post.comments?.length === 1 ? "تعليق" : "تعليقات"}. إضافة currentUserProfile إلى PostCardProps وتمريره إلى CommentsDialog.
